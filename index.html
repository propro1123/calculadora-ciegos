Bienvenido

Este es el sitio del Proyecto PCPC, una calculadora diseñada para personas con discapacidad visual, combinando tecnología accesible, bajo costo y facilidad de uso.

Planteamiento del Problema

Una calculadora para ciegos que busca facilitar el uso de operaciones matemáticas de manera autónoma, sin depender de terceros ni de costosos equipos especializados.

Planteamiento del Problema El acceso a dispositivos cotidianos como una calculadora representa un desafío significativo para personas con discapacidad visual, ya que la mayoría de los modelos comerciales dependen del reconocimiento visual del teclado y la pantalla. Esta situación limita la autonomía al realizar cálculos sencillos o complejos y demanda soluciones asequibles y accesibles. 1. Limitaciones de los dispositivos actuales Algunos dispositivos asistivos, como los calculadores parlantes (talking calculators), existen en el mercado; reproducen cada tecla al presionarla y anuncian el resultado en voz, pero suelen ser costosos o con funciones básicas brighthubeducation.com . Las aplicaciones tradicionales en teléfonos o en la web frecuentemente no aceptan entrada desde dispositivos braille o son incompatibles con un uso no visual pleno Reddit +1 . 2. Avances en tecnologías multimodales En otras áreas de las tecnologías asistivas, se reconoce que la combinación de audio, tacto y retroalimentación háptica (vibraciones, texturas) mejora sustancialmente la experiencia del usuario PMC +1 MDPI . Por ejemplo, aplicaciones con detección de objetos para usuarios ciegos incorporan retroalimentación háptica y auditiva para transmitir información no visual MDPI . 3. Necesidad de soluciones inclusivas y económicas Proyectos como "BrailleTap", que convierte gestos táctiles en cálculos usando braille y audio, han mostrado mejor velocidad y accesibilidad en pilotos iniciales frente a calculadoras táctiles tradicionales ResearchGate . En el campo educativo, se han identificado técnicas multisensoriales (como TouchMath o aplicaciones especializadas) que favorecen el aprendizaje matemático mediante tacto y voz Europe PMC . 4. Propuesta: una solución accesible, inclusiva y autosuficiente En este contexto, el Proyecto PCPC propone desarrollar una calculadora física que permita: Entrada táctil mediante un teclado adaptado con etiquetas en braille. Visualización accesible, con una pantalla LCD I2C que provee feedback visual para personas con baja visión o sus acompañantes. Retroalimentación auditiva, mediante reproducción de voz clara de cada número y operación, sin depender de internet ni dispositivos externos. Portabilidad y costo contenido, usando componentes económicos y de fácil acceso como Arduino Nano, DFPlayer Mini, parlantes o auriculares estándar, y microSD de 32 GB. Este enfoque busca no solo ofrecer una herramienta funcional, sino también promover inclusión real, permitiendo que estudiantes, trabajadores o cualquier persona con discapacidad visual realice cálculos de manera independiente, eficiente y accesible en diversos contextos.

Materiales

    Arduino Nano

    Teclado Matricial 4x4 adaptado con braille

    Utilizamos un TECLADO MATRICAL 4x4 para poder códigificar la parte interna de la calculadora.

    La descripción del mismo:

    El teclado matricial 4x4 está formado por una matriz de pulsadores dispuestos en filas (L1, L2, L3, L4) y columnas (C1, C2, C3, C4), con la intención de reducir el número de pines necesarios para su conexión.

    Pantalla LCD 16x2 con módulo I2C

    La pantalla LCD sera la encargada de mostrar los números a medida que se vayan tocando en el teclado matrical 4x4.

    Descripción del mismo:

    Pantalla LCD Una pantalla de cristal líquido, o LCD (sigla del inglés liquid-crystal display), es una pantalla plana y delgada formada por un número de píxeles en color o monocromos colocados delante de una fuente de luz o reflectora.

    El módulo I2C irá conectado a la pantalla LCD.

    Descripción del mismo:

    La I2C interfaz LCD 1602 PCF8574 es un módulo que permite la comunicación simplificada entre un microcontrolador y una pantalla LCD1602 o LCD2004. A través del protocolo I2C (Inter-Integrated Circuit).

    DFPlayer Mini

    Tarjeta microSD de 32GB

    Descripción del mismo:

    Las tarjetas microSD suelen utilizarse para ampliar la capacidad de almacenamiento de teléfonos inteligentes, drones, equipos de juegos y cámaras. Los dispositivos de hardware también son retrocompatibles con las tarjetas microSD, al igual que con las tarjetas SD.

    Parlante o auriculares

    Cables y protoboard

    Fuente de alimentación

    Como fuente de alimentación utilizamos una batería de litio, especificamente la siguiente: "Batería Lsm 18650 2000mah Recargable 3.7v Litio".

    Descripción de la misma:

    Batería Yeti Tank 1400/3000 5Kw V2 Es la batería más potente por su tamaño, la llamada 18650 de Goal Zero es fabricada de litio y tiene en ambos lados polos negativos y positivos.

Descripción del Proyecto

El Proyecto PCPC nació como respuesta a una necesidad concreta: permitir que personas con discapacidad visual puedan realizar cálculos matemáticos de manera autónoma, sin depender de costosos dispositivos del mercado.

El dispositivo combina botones con etiquetas en braille, una pantalla LCD I2C y un módulo de audio que reproduce la voz mediante archivos almacenados en una tarjeta microSD.

          // LCD I2C (dirección 0x27, 16 columnas, 2 filas)
          LiquidCrystal_I2C lcd(0x27, 16, 2);
          
          // DFPlayer - conexión por SoftwareSerial
          SoftwareSerial mySerial(10, 11); // RX, TX del Arduino hacia DFPlayer
          DFRobotDFPlayerMini player;
          
          // Teclado 4x4 sin librería
          const byte filas[4] = {9, 8, 7, 6};    
          const byte columnas[4] = {5, 4, 3, 2}; 
          
          char teclas[4][4] = {
            {'1','2','3','+'},
            {'4','5','6','-'},
            {'7','8','9','*'},
            {'C','0','=','/'}
          };
          
          void setup() {
            Serial.begin(9600);
          
            // LCD
            lcd.begin(16, 2);
            lcd.backlight();
            lcd.setCursor(0, 0);
            lcd.print("Calculadora");
          
            // DFPlayer
            mySerial.begin(9600);
            if (!player.begin(mySerial)) {
              lcd.clear();
              lcd.print("Error DFPlayer");
              while (true);
            }
            player.volume(25); // Volumen de 0 a 30
          
            // Pines del teclado
            for (int i = 0; i < 4; i++) {
              pinMode(filas[i], OUTPUT);
              digitalWrite(filas[i], HIGH);
            }
            for (int i = 0; i < 4; i++) {
              pinMode(columnas[i], INPUT_PULLUP);
            }
          
            delay(1000);
            lcd.clear();
          }
          
          void loop() {
            for (int f = 0; f < 4; f++) {
              digitalWrite(filas[f], LOW);
              for (int c = 0; c < 4; c++) {
                if (digitalRead(columnas[c]) == LOW) {
                  char tecla = teclas[f][c];
                  lcd.setCursor(0, 0);
                  lcd.print("Tecla: ");
                  lcd.print(tecla);
                  reproducir(tecla);
                  delay(300); // Antirrebote
                  lcd.clear();
                }
              }
              digitalWrite(filas[f], HIGH);
            }
          }
          
          void reproducir(char tecla) {
            int num = 0;
            switch(tecla) {
              case '0': num = 1; break;
              case '1': num = 2; break;
              case '2': num = 3; break;
              case '3': num = 4; break;
              case '4': num = 5; break;
              case '5': num = 6; break;
              case '6': num = 7; break;
              case '7': num = 8; break;
            }
            if (num > 0) {
              player.play(num);  // Reproduce 0001.mp3 hasta 0016.mp3 según número
            }
          }
